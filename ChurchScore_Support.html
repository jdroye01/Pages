Getting Started

Setup Guide
ℹ️ ChurchScore works on a single iPad with no setup at all. Only follow these steps if you want to sync across multiple iPads.
1
Install the sync server
Download the ChurchScore server installer for your platform (Linux, macOS, or Windows) and run the one-line install command. It automatically installs Node.js, PostgreSQL, and registers the server as a background service.
2
Install your VPN of choice on the server and each iPad
Download your VPN of choice on your server machine and sign in. Install the your VPN's app on each iPad and sign in with the same account. Note your server's your VPN IP — it may start with 100.x.x.x.
3
Enter the server URL in ChurchScore
Open ChurchScore on each iPad → Settings → Live Sync → Edit → enter ws://100.x.x.x:8080 (your VPN IP) → Save.
4
Pull a full sync
Once connected (you'll see a green "Connected" status), tap Pull Full Sync from Server to download any existing data onto the new iPad.
5
Add your classes, members, and actions
Go to Settings to add your classes first, then members (assigned to a class), then actions (the activities that earn points). You can also import members from an Excel or CSV file.

Multi-iPad Sync

Real-Time
💡 Real-time sync requires all iPads and the server to be connected to the same VPN network. The sync status indicator in Settings → Live Sync shows the current connection state.

There is no hard limit. The server supports as many simultaneous WebSocket connections as your server hardware can handle. For typical church use, 10–20 iPads is well within any reasonable server's capacity.

The app continues to work normally. All changes are saved to the local SQLite database on the iPad immediately, regardless of server connectivity. The status indicator will show "Offline — changes queued."

When the server comes back online, all queued changes are automatically sent in order. No data is lost.

Open a browser on any device connected to your VPN network and visit http://100.x.x.x:8080/health (replace with your server's VPN IP). You should see a JSON response showing "status": "ok" and the number of connected clients.

On the server itself, run systemctl status churchscore (Linux) or check Services (Windows).
